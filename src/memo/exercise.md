# Exercise: More memoization
The `memo` example has the following setup:

*   Clicking the button "My Chart N" (where N starts as 1) _prepares_ a new chart, for which data can be generated.

*   Clicking the button "Generate data" generates data and renders a "complex" chart, a typically slow process.

Currently, preparing a new chart takes a really long time, and it also seems to generate data, which is an error (data should only be generated by clicking the "Generate data" button).

Fix the problem of preparing a new chart by profiling the code (using the Chrome Profiler and/or React Profiler) to identify the bottleneck, and apply React memoization techniques (`React.memo()`, `useMemo`).

When the code functions properly, clicking the "My Chart N" button should _instantly_ increase N by 1, and new data should _only_ be generated when clicking the "Generate data" button.

### Optional
As an alternative to memoization, there is a simple approach that solves the problem, and it involves the _state colocation_ principle. Try to apply it to the original code to achieve the same results, i.e. _without_ any use of `React.memo()` and `useMemo`.
